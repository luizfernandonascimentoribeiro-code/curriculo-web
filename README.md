# curriculo-web
Link do Currículo: https://luizfernandonascimentoribeiro-code.github.io/curriculo-web/
1. Estrutura da Página – HTML5

O HTML5 foi utilizado para criar a estrutura do site, organizando o conteúdo em seções como dados pessoais, formação acadêmica, habilidades, projetos e contato.

Principais elementos utilizados:

* <header> para o cabeçalho da página;
* <section> para organizar os conteúdos;
* <h1> até <h3> para títulos;
* <p> para textos;
* <ul> e <li> para listas;
* <form> para o formulário de contato.

O HTML funciona como o esqueleto da aplicação, definindo quais informações serão exibidas ao usuário.


2. Estilização da Interface – CSS3

O CSS3 foi utilizado para personalizar a aparência do currículo, tornando o site mais moderno e agradável visualmente.

Foram aplicadas técnicas como:

* Definição de cores;
* Ajuste de fontes;
* Espaçamentos e alinhamentos;
* Responsividade para dispositivos móveis;
* Efeitos visuais em botões e menus.

O CSS é responsável pelo design da página, separando a apresentação da estrutura criada em HTML.


3. Interatividade – JavaScript

O JavaScript foi utilizado para adicionar funcionalidades dinâmicas ao site.

Exemplos de recursos implementados:

* Alternância entre tema claro e escuro;
* Resposta a cliques em botões;
* Manipulação de elementos da página;

O JavaScript permite que o site responda às ações do visitante sem necessidade de recarregar a página.


4. Processamento de Dados – PHP

O PHP foi utilizado para demonstrar o desenvolvimento do lado do servidor.

Sua principal função no projeto foi:

* Receber informações enviadas pelo formulário de contato;
* Processar os dados enviados pelo usuário;
* Exibir mensagens de confirmação.

O PHP é uma linguagem de programação executada no servidor, muito utilizada para sistemas web e aplicações dinâmicas.


5. Controle de Versão – Git e GitHub

Para armazenar e disponibilizar o projeto, foi utilizado o GitHub.

Etapas realizadas:

1. Criação de um repositório;
2. Envio dos arquivos do projeto;
3. Organização do código-fonte;
4. Publicação do site através do GitHub Pages
